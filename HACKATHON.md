# Portaldot Online Mini Hackathon S1

**Organizer:** Portaldot
**Platform:** DoraHacks
**Format:** Virtual / Online
**Page:** https://dorahacks.io/hackathon/portaldot-online-s1

## Overview

A lightweight online hackathon for developers to quickly build a prototype, validate an idea, and showcase their work. The emphasis is on **low-barrier participation, runnable MVPs, and clear demos** — not a polished, production-ready commercial product.

Projects must be built on **Portaldot**, a **Substrate-based, Rust-first, non-EVM** chain, and use **POT as gas**.

## Tracks

### 1. Native Onchain Apps
User-facing dApps built natively on Portaldot — consumer apps, payment flows, mini dApps, community products, creator tools, membership systems. Embrace the native Substrate/Rust architecture rather than EVM-based assumptions.

### 2. Builder Tools for Portaldot
Developer-experience tooling that reduces friction in a Substrate-based, Rust-first environment — local dev helpers, CLIs, explorers, monitoring tools, testing utilities, deployment scripts, debuggers, docs helpers, and workflow automation.

### 3. Onchain Identity & Coordination
Products that help users, teams, and communities manage identity, reputation, permissions, credentials, and collaboration — onchain profiles, reputation systems, contribution records, badges, credentials, attendance proofs, role-based access control, membership systems, and community coordination tools.

### 4. AI-Powered Onchain Workflows
AI- or automation-driven workflows on Portaldot — dev copilots, transaction assistants, workflow agents, chain-data analysis tools, monitoring/alert systems, and user-facing products combining AI interfaces with Portaldot-based execution. Focus on one strong, demonstrable workflow loop.

**Every track expects:** a clear scenario / pain point, a runnable MVP, real Portaldot integration, and a strong end-to-end demo.

## Judging Criteria

1. **Portaldot Native Deployment** (mandatory) — must deploy using Portaldot's features and use native POT for gas; cannot be copied from other sources.
2. **Demo Completion** — a functioning MVP that is demonstrable.
3. **Application Value** — the project must have market potential.
4. **Presentation Quality** — clear, logical live demo with key points highlighted.
